## 1. 프로젝트 개요

- **구분**: 대규모 팀 프로젝트 (프론트엔드 담당)
- **기여도**: 프론트엔드 개발 기여도 20% (전체 11명 대규모 구성: 프론트엔드 5명, 백엔드 4명, 기획 1명, 디자인 1명)
- **기술 스택**: React, JavaScript, Styled-components, Axios, React Router, Git
- **기간**: 2024.01 ~ 2024.02

---

## 2. 대규모 협업 및 기술적 성과

### 🤝 다자간 애자일 협업 프로세스 체득
- **배경**: 기획(PM), 디자인, 백엔드, 프론트엔드 5명 등 총 11명이라는 대규모 인원이 한 달 남짓의 기간 동안 제품을 완성해야 했습니다.
- **도전 과제**: 다수의 프론트엔드 개발자 간 소스코드 충돌을 조율하고, 백엔드와의 API 사양서 정합성을 빈틈없이 관리해야 했습니다.
- **해결 전략**:
  - **Git Flow 정형화**: 기능 단위 분리 머지 전략과 통일된 커밋 메시지 컨벤션을 명문화하고 엄격히 이행하여 Git 히스토리 꼬임을 전면 차단했습니다.
  - **피어 리뷰 활성화**: 프론트엔드 파트원 간 데일리 코드 리뷰 및 기능 컴포넌트 구조 통일화를 시도하여 공통 컴포넌트의 중복 생성을 예방하고 재사용률을 제고했습니다.

### 📐 컴포넌트 중심 설계와 API 연동
- **도전 과제**: 다양한 여행 루트와 장소 데이터를 동적으로 매끄럽게 핸들링해야 했습니다.
- **해결 전략**:
  - `Styled-components`를 적극 활용하여 테마 기반의 반응형 디자인 요소를 구현했습니다.
  - `Axios`를 통한 Rest API 통신을 공통 HTTP 모듈로 추상화하여 중복 토큰 주입 로직을 간결히 정리하고, 화면 렌더링 성능 최적화를 구축했습니다.
