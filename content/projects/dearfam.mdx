## 1. 프로젝트 개요

- **구분**: 팀 프로젝트 (프론트엔드 리드 / 기획 참여)
- **기간**: 2024.11 ~ 2025.09
- **기여도**: 프론트엔드 개발 기여도 80% (팀 구성원: 프론트엔드 2명, 백엔드 1명, 디자이너 & 기획 2명)
- **기술 스택**: Flutter & Dart (초기 MVP) &rarr; React & TypeScript (웹 마이그레이션), Zustand, TanStack Query, Tailwind CSS, Shadcn UI, Firebase, Riverpod (Flutter)
- **주요 성과**: 
  - 2024년 클라우드 아이디어 공모전 **대상 (부산시장상)** 수상
  - Pre-스타트업 선정으로 창업 사무실 입주 지원 확보
  - 제 15회 숭실캡스톤디자인 경진대회 **장려상** 수상

---

## 2. 핵심 성과 및 기술적 도전

### 🔄 MVP 기술 스택 마이그레이션 (Flutter &rarr; React)
- **배경**: 초기 검증 속도를 위해 하이브리드 앱 프레임워크인 Flutter와 Firebase 기반으로 빠르게 MVP를 구축하였습니다. 이를 토대로 '2024 클라우드 아이디어 공모전 대상(부산시장상)'을 수상하고 창업 입주실 혜택까지 확보하였습니다.
- **도전 과제**: 시장성 검증 이후, 서비스의 웹 접근성을 강화하고, 복잡한 레이아웃 확장성 및 검색 엔진 최적화(SEO)를 달성하기 위해 프론트엔드 스택을 React/TypeScript 웹 환경으로 전면 마이그레이션하기로 결정했습니다.
- **해결 전략**: 
  - Flutter의 Riverpod 상태 관리를 가볍고 직관적인 `Zustand`로 전환하여 전역 상태를 동기화했습니다.
  - 비동기 데이터 패칭 및 데이터 유효성 캐시 라이프사이클 관리를 위해 `TanStack Query`를 도입하여 불필요한 중복 API 요청을 완전히 제거하고 컴포넌트 간 통신을 최적화했습니다.

### 📐 페이지 중심(Page-Centric) 아키텍처 설계
- **배경**: 프로젝트 피처 수가 점진적으로 확대됨에 따라, 컴포넌트와 커스텀 훅들이 엉키며 Git 충돌 및 유지보수 비효율이 발생하기 시작했습니다.
- **해결 전략**:
  - `pages/` 폴더 하위에 개별 페이지 전용 `components/`와 `hooks/`를 격리하는 폴더 분리형 아키텍처를 수립했습니다.
  - 관련 로직의 응집도를 높여 사이드 이펙트 범위를 제한하고, 협업 시 동일 파일 수정에 따른 코드 충돌을 최소화하여 생산성을 크게 끌어올렸습니다.

### Mock Service Worker (MSW) 도입을 통한 병목 해결
- **배경**: 백엔드 API 설계 완료 후 실제 구현 지연에 따른 프론트엔드 작업 지연이 자주 발생하여, 프론트엔드가 백엔드 의존성 없이 개발할 수 있는 환경이 시급했습니다.
- **해결 전략**:
  - `MSW(Mock Service Worker)`를 전면 도입하여 네트워크 레벨에서 목(Mock) API를 모킹했습니다.
  - 이를 통해 백엔드 완성 전에도 API 인터페이스 사양서만으로 완전히 독립적인 UI 개발과 프론트엔드 테스트 환경을 독자적으로 구축하고, 개발 주기를 단축시켰습니다.

### 🎨 일관된 컴포넌트 디자인 시스템 구축
- **배경**: 여러 명의 개발자가 각자 UI를 개발함에 따라 버튼, 모달, 입력 폼 등의 미세한 마진 및 레이아웃 불일치 현상이 발생했습니다.
- **해결 전략**:
  - `Tailwind CSS`와 `Shadcn UI`를 기반으로 재사용 가능한 공통 디자인 시스템 컴포넌트를 설계했습니다.
  - 이를 통해 개발 속도를 대폭 가속하고, 디자인 가이드라인 수정을 일원화하여 높은 UI 일관성을 확보했습니다.
